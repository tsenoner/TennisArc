# Issue #25 — Reproducing Tennis Abstract: ELO, yELO, Ranking Points & W/L

> Canonical findings summary for issue #25. Supersedes and consolidates the scattered working docs
> (`docs/elo-investigation-findings.md`, `docs/elo-formula.md`, `docs/elo-methodology.md`,
> `docs/yelo-reproduction.md`, `docs/points-reproduction.md`). Only **current** findings are folded in;
> every value flagged `superseded` in those docs has been dropped. Repo match scope: **ATP ≥ 2009, WTA ≥ 2015.**

## Executive summary

This issue reverse-engineered the four published Tennis Abstract (TA) products — the career **Elo** board, the season **yElo** board, ranking-**points**-earned, and **W/L** counts — from the public Sackmann match CSVs and archived TA boards. The methods are now recovered to the point that:

- **Elo (full board):** update rule fully recovered — `D=400`, single K-law `K = 250/(n+5)^0.4`, surface rating = exactly `0.5·overall + 0.5·surface_raw`, single low-1200s entrant seed, RET/WO **not** counted. A from-scratch engine reproduces today's board to ~**3 Elo (ATP) / ~1.4 Elo (WTA)**; **board-to-board replay** (seed from TA's own Wayback board, replay one window) is **byte-exact in clean windows** (per-transition median 0.04–0.10 Elo).
- **yElo (season):** Sackmann's 2021 method fully recovered. Reproduced to **med-of-medians ATP ~5.4 / WTA ~5.9 Elo**. Byte-exact is **proven impossible** from public data (two independent proofs).
- **Ranking points:** per-tournament tier+round points reproduce **EXACTLY** (Djokovic 2023 = 11245 exact). Year-end **TOTAL** coverage is **306/833 top-30 rows exact** (up from 151). Remaining gaps are **structural**, not bugs.
- **W/L counts:** **cannot** reach 100% — proven irreducible from TA's own data. Ceiling **ATP ~94.9% / WTA ~68.3%**; the one real fix shipped was adding level `W` to the WTA tour set (+181 W/L).

The recurring theme: **you cannot out-compute the source.** TA publishes weekly, 1-decimal boards generated by a private daily Elo over a privately-revised match DB. Where we have TA's own board as a seed we hit byte-exactness; where we must regenerate from scratch we hit a hard, *provable* floor.

## 1. ELO — full career board

### Solved method (confirmed constants)

| Element | Value | Confidence |
|---|---|---|
| Win expectation | `E = 1 / (1 + 10^((rOpp − rSelf)/400))`, base **D=400** | Confirmed (TA "Intro to Tennis Elo" 2019-12-03; D weakly identified but 400 inside every CI) |
| K-factor | **`K = 250/(n+5)^0.4`**, single law both tours, `n` = player's prior counted-match count, no cap | Confirmed — 250 maximizes byte-exact replay windows; implied-272 read rejected as single-window leakage; exponent 0.4 (not 0.72) |
| Surface rating | **exactly `0.5·overall + 0.5·surface_raw`**, every surface, every sample size; null if 0 matches on that surface | Confirmed — free OLS → a=0.5000, b=0.5000, R²=1.0000 over ~75k obs; archive-confirmed on 2019-12-02 board |
| Entrant seed | **single ~1155 (ATP) / ~1200 (WTA)**, NO level/qualifying split (TA's published "low 1200s") | Confirmed by board-replay; the low seed cancels the inflation from adding Challengers/qualifying |
| Walkovers / RET | **NOT counted** in Elo updates | Confirmed — single-match WO/RET players move only 12% (ATP)/24% (WTA) vs 78–79% for completed matches |
| Margin of victory | **Not used** (binary S = 1 / 0) | Confirmed — TA 2025-11-28 "Elo looks at match results — period" |
| Idle players | **Frozen** between adjustments, no decay | Confirmed — median Δ = 0 even across the 175-day COVID gap |
| Absence dock | discrete ~**−100** round (multiple-of-5) **DISPLAY** dock on currently-absent players rated **≥1900** after ~8 wk idle; dropped on return; range −90…−115; occasionally extended a step toward −150; rarely refunded by a round +100/+105 around return | Confirmed mechanism (a semi-manual display overlay, NOT a history-baked curve) |
| Board eligibility | listed iff **≥10 counting matches in a trailing 52-week rolling window**; singles only, doubles excluded | Confirmed (report-page footnote) |

**Match-inclusion scope**
- **ATP:** Grand Slam (G) / Masters (M) / ATP-level 250–500 (A) / Finals (F) / Davis-team (D) / Challenger main draw (C) + tour-level qualifying. Excludes ATP Futures (S), exhibitions, juniors.
- **WTA:** G / WTA1000–Premier (PM/P) / International–250 (I) / Finals (F) / BJK-team (D) + qualifying + **ITF ≥ $50K only** (tiers 50/60/75/80/100/125 counted; 15/25/35/40 NOT). Excluding sub-$50K ITF + WO/RET is what drops the WTA whole-span median-of-medians from 7.3 → 1.4.

**Player-ID fragmentation:** duplicate Sackmann player_ids (a multi-feed data-entry artifact, ATP issue #154 / WTA #41) are resolved by a **dominant-id join** in `computeRatingsAsOfSorted` (shared fullKey → id with ≥4× runner-up's match count). 19 of 8207 ATP fullKeys collide; Mensik & Landaluce appeared in 2026 draws. **Always join on id, never bare surname.**

### Accuracy achieved
- **From-scratch engine:** reproduces today's live board to **~3 Elo (ATP) / ~1.4 Elo (WTA)**.
- **Board-to-board replay** (`board(T)` is truth, replay one window → `board(T+1)`): per-transition **median-of-medians 0.10 (ATP) / 0.04 (WTA)** all-players; **271/332 (ATP) & 181/230 (WTA)** transitions ≤1 Elo. Restricted to players who actually played a window match: ~**1.4 (ATP) / ~1.0 (WTA)**. Many windows are byte-exact (one quiet window froze 441/444 players within 0.1). Matches processed in true round order (Sackmann lists finals first; `lib.ts:roundRank`). Dense Wayback captures = 338 ATP / 240 WTA board dates (median gap 7 days).

### Proven irreducible (and why)
- **Byte-exact from-scratch is impossible.** Three unrecoverable unknowns: the exact per-tour seed (published only as "low 1200s"), the semi-manual round-number injury-dock schedule, and whether K was re-tuned since 2016. TA's generation code is **private** (`JeffSackmann/tennis_viz/players_weekly_elo.py` carries verbatim "historical ratings and code to generate ratings are not public").
- **A mid-2018 full-history recompute discontinuity** exists in TA's published history: when TA expanded inclusion ("20 tour-level matches" → "10 matches incl. Challengers/qualifying/ITF $50K+") it re-ran its whole history, shifting the scale one-time **~−82 median (~−230 at the top)** at **ATP 2018-06-11 / WTA 2018-05-28** (Federer −235, Serena −391). An era boundary, not a monthly update; board-replay excludes it.

## 2. yELO — season board

### Solved method
For each player X: **reset to 1500** and match-count `n=0` on Jan 1; replay X's current-calendar-year matches in **play order**; each opponent stays at their **REAL full-Elo rating at match time** (NOT reset); **only X updates**. Same update rule as the full board (`D=400`, `K=250/(n+5)^0.4`). Sackmann's 2021-03-16 "Repurposing Elo for streaks, seasons…" method. (Muguruza example: starts 1500, beats Mladenovic@1817 → 1605.)

- **Listing threshold = ≥3 career/season wins** (the page's "five or more wins" prose is stale boilerplate; the ≥3 set matches both 2026 ATP boards exactly, 0 missing/0 extra).
- **Inclusion:** tour-level main draw + tour-level qualifying + Challenger/WTA-125 (level C) main draw + ITF $50K+ main draw. **Excludes** Challenger/ITF qualifying (except WTA-125 qual), pure walkovers, sub-$50K ITF, Olympics. Tour asymmetry: **WTA counts WTA-125 (level C) qualifying; ATP does not count Challenger qualifying** (provable from Milos Karol 2025: board 21 = 57 − 36 qual).
- **Season attributed by tournament END year** (late-December openers — United Cup, Brisbane, Hong Kong — count in the new year).
- **Reproduction reads opponents off TA's own published full-Elo boards, linearly interpolated to `m.playDate`** (`oppRatingAt` in `yelo-fit.ts`). Two free accuracy wins: (1) interpolate at estimated play date not tournament start, (2) never-on-any-board opponents → **flat 1325** (`OFF_BOARD_SEED`). Below-floor players fall through to an anchored forward-Elo pass (`tlSeed=1500`).
- **Feed de-dup** is mandatory once 125-qualifying is in scope: the WTA qualifying/challenger feed re-lists every level-C match twice (1343 byte-identical rows in 2026); `loadMatches` drops dups keyed by `tourneyId|round|winnerId|loserId|score`.

### Accuracy achieved
- **Med-of-medians ATP ~5.4 / WTA ~5.9 Elo** after the two final fixes (was 7.0 / 8.4). Byte-exact board counts ATP 24→49, WTA 5→35; Alcaraz 2026 byte-exact (2124 vs 2124.4).
- **W/L tally:** aggregate ATP 95% (10201/10730), WTA 68% (8609/12690); 2023 boards reach ATP 100% / WTA 98%.

### Proven irreducible (two independent proofs)
1. **Board granularity:** TA's yElo uses each opponent's internal ~daily full-Elo; only ~weekly 1-decimal boards are public. Interpolating weekly boards can't pin an opponent finer than ~2–4 Elo. Capture density is already maxed (21-day Feb cadence is TA's real publication interval).
2. **Opponent match-data drift:** via TA's own `jsmatches`, TA's match set differs from Sackmann's for many players — we can't even reproduce exact W/L (the ATP 94.9% / WTA 68.3% ceiling). If opponent results aren't exactly right, their reconstructed Elo can't be either.

A **full-precision yElo engine was built and tested** (`.scratch/elo-reverse/yelo-engine.ts`): board-anchored replay, full precision between captures, all opponent models (frozen-at-entry, live-climbing, 50/50 blend, pure from-scratch) — **all worse** than interpolation@playDate. *"You cannot out-compute the source."*

## 3. Ranking POINTS — points-earned reproduction

### Solved method
Engine is **spec-driven** (`ingest/points/engine.ts` + `POINTS-TABLES.md` + `TIER-LISTS.md` JSON blocks as single source of truth): loads era-correct per-round point tables, an ordered tier ruleset, per-year tier lists, and best-N rules, resolving by season. Round-reached extraction is exact (champion→W, runner-up→F, …) plus ATP/WTA Finals round-robin accumulation (per-RR-win + SF/F bonuses).

Key rules recovered:
- **BYE RULE** (ATP rulebook §9.03 G.2, identical 2019 & 2023): a seeded player who reaches R2 on a bye and then loses is scored as a **first-round** loser, not at his actual exit round. This was the final fix — took ATP-2019 from 0 → 12/30 exact and made Alcaraz 2023 exact.
- **ATP Tour Finals is Sackmann level `A`** ("Tour Finals"), **NOT `F`** (level `F` = NextGen Finals = 0 pts). **Detect by name.** Finals scoring: 200/RR-win + 400 SF + 500 F, 1500 if undefeated champion.
- **Qualifying points add on top** of the main-draw result (carve-outs: a non-playing "qualifier" gets only MD points; a qualifier losing MD-R1 keeps Q points).
- **Olympics award ranking points only in 2012 (ATP); 2016/2021/2024 = 0; WTA all years = 0.**
- **ATP 500 / WTA tier resolution uses name+year override lists, NOT tourney_level/draw_size** (e.g. Dallas/Doha/Munich are 250 through 2024, 500 only from 2025).
- **Best-N counting caps** are era- and tour-specific (ATP A 2009-23: core 18 + Finals bonus; ATP B 2024-25: core 19+; WTA 2014-23: core 16 + Finals bonus; WTA 2024-26: core 18+).
- **Per-round point tables**: ATP Era A (2009-2023, byte-identical to 2019) vs Era B (2024-26 raised runner-up curves); WTA 2014-2025 with adversarially-confirmed corrections (WTA 500 W=470 pre-2024→500; WTA 250 W=280 pre-2024→250).

### Accuracy achieved
- **Per-tournament tier+round points: EXACT.** Djokovic 2023 = 11245 exact; Alcaraz 2023 = 8855 exact; Nadal/Federer/Shapovalov 2019 exact.
- **Year-end TOTAL coverage: 306/833 top-30 rows exact** (was 151) after: yearRange era-key **regex bug fix** (`\b` fails on underscore-joined keys) + ATP **Wimbledon-2022 ZERO** + **2012-Olympics medal points** + ATP **World Team Cup exclusion** + name-joins.

### Proven irreducible / structural (not bugs)
1. **52-week rolling window:** the year-end **ranking** carries ~6 weeks of the prior year's indoor results that a calendar-year sum excludes — a universal **~±110 residual** (Medvedev 2019 +180 = exactly this carry). Validating against the year-end **RACE** (pure calendar year) would remove it, **but we don't have a Race dataset.**
2. **Rank-scaled team events** award points by opponent ranking + stage, not round reached, so they're not derivable from match CSVs: **United Cup** (2023+, max 500), **ATP Cup** (2020-2022, max 750), **Davis Cup**. Proven: every 2023 ATP top-8 under-counter (Tsitsipas, Zverev, Fritz, Tiafoe, Norrie, Hurkacz) played United Cup; non-participants (Alcaraz, Sinner) are exact.
3. **COVID-frozen rankings** (~22 months): **ATP/WTA 2020-21 = 0 reproducible** from a single calendar year.

## 4. W/L — yElo-board win/loss-count reproduction (Phase 1)

### Solved method & outcome
Tally W/L per player against the yElo board's published counts. The one real fix shipped: **add level `W` to the WTA tour set (+181 W/L)**. ITF-qual / walkover / Olympics-exclude experiments **all regressed**.

### Proven irreducible
Cannot reach 100% — **proven from TA's own data**: (a) some Wayback captures are **outright corrupted** with doubled W/L counts (WTA 20211227 & 20240101 — Muguruza 84-32 = 2× her real 42-16); (b) **since-revised states** (the 20260420 WTA capture: 26 of 75 under-counters are mathematically impossible at any cutoff — board W or L exceeds the whole-season match count); (c) **Sackmann↔TA match-set drift**. Ceiling **ATP ~94.9% / WTA ~68.3%**, and we hit it.

## What's next

The methodology is solved; the remaining work is to stop being limited by the *source's* publication granularity and revision behavior by building our **own** complete data universe and engines, plus closing the still-open **storage** half of #25. See the follow-up issues:

1. **Own data-acquisition pipeline** — fetch all Elo-feeding matches directly from official ATP/WTA JSON (results + W/L + per-tournament points), Sackmann CSV as backbone/fallback, continuous + incremental + append-only.
2. **Perfect ranking-points engine** — own complete data → exact year-end totals by sourcing team-event points and the Race/rolling-window data that block exact reproduction today.
3. **Own Elo engine** — built on the confirmed `D=400` / K-law / 50-50 surface / single-seed / no-RET rules over our own match universe; retire the superseded split-seed + smooth-injury model.
4. **Data restructuring** — move from static per-slam JSON to a DB / derived-artifact pipeline (the open storage half of #25).
