# Elo methodology

TennisArc ships surface-aware Elo on every player, recomputed from [Jeff Sackmann's](https://github.com/JeffSackmann) match CSVs and **frozen at each slam's start date** (a 2016 draw shows 2016 Elo, not today's). The engine lives in `ingest/historical-elo.ts`; the per-tour fitted parameters in `ingest/elo-config.ts`; the calibration harness in `ingest/calibrate-elo.ts`; the regression guard in `ingest/historical-elo.fixture.test.ts`.

The goal is to reproduce [Tennis Abstract's published board](https://tennisabstract.com/reports/atp_elo_ratings.html) as closely as public data allows. This investigation (issue #25, 2026-06-14) verified TA's methodology against Sackmann's own blog/code and **measured** the result against the live board.

## Byte-exact reproduction is impossible — and that's settled

Sackmann's own repo (`tennis_viz/players_weekly_elo.py`) states verbatim: *"(historical ratings and code to generate ratings are not public)."* The published board is generated by code we cannot see: the entrant **seed** is never published (only "low 1200s"), the injury **dock** publishes only two endpoints (100 @ 8wk, 150 @ ~1yr — shape between them unstated), and the on-return **K-multiplier** schedule is path-dependent and unrecoverable. So the original "±1–2 Elo, ideally exact" target is **retired**. We target a documented-methodology approximation and pin the achieved accuracy with a fixture.

For the **current** slam, the exact published numbers remain available by scraping (`ingest/elo.ts`, byte-correct). The recompute exists only to *rewind* to historical events on the same scale.

## The parameters

| Driver | Value we use | Source / confidence |
|---|---|---|
| Win expectation | `1 / (1 + 10^((rB − rA) / 400))` | TA "Introduction to Tennis Elo" — verbatim. Exact. |
| K-factor | `250 / (n + 5)^0.4` (per player, own prior count) | FiveThirtyEight 2016 tuning Sackmann adopted. Form documented; current value unconfirmed (0.72). |
| Margin-of-victory | **none** (binary win/loss) | TA 2025-11-28: *"Elo looks at match results — period."* Exact (0.97). |
| Surface Elo | flat `0.5·overall + 0.5·pureSurface`, every surface, every sample size; `null` if 0 surface matches | TA report page + Heavy Topspin 2017/2019: *"50/50 worked for each surface."* (0.92) |
| Match inclusion | tour-level + tour-level qualifying + Challenger main draw (ATP); + ITF ≥ $50K (WTA). Walkovers/retirements **counted** (Sackmann lists a winner). | TA 2025-11-28, verbatim scope (0.95). Davis Cup/team `D` included. Excludes sub-$50K ITF, exhibitions, juniors. |
| Entrant seed | **fitted** per tour: ATP `seedTour 1450 / seedSub 1130`, WTA `1350 / 1050`. Sub-tour = Challenger (`level C`) or qualifying (`round Q*`). | TA: *"a number in the low 1200s … depends a bit on tournament level and gender"* — value never published, so empirically fit for the 1968 history (0.93). |
| Injury/absence dock | **yes** — a currently-absent player rated ≥1900 is docked ~100 Elo at 8 active-season weeks out, rising to ~150 at ~1 year (`TA_LAYOFF_DOCK`, applied at extraction). | TA's **published board already docks** absent players: the penalties *"bring down the current Elo ratings of the players who are in the middle of long breaks … giving us ranking tables that come closer to what we expect"* (2018); during COVID he had to explicitly **suspend** that penalty board-wide (2021). 100/150 endpoints verbatim; ≥1900 gate verbatim (0.9). |
| History start | **1968** (full Sackmann history) | Tour-level exists 1968+ (Challengers 2008+, qualifying 2011+, so the deep past is tour-level only). The deeper burn-in matures the scale; the seed is re-fit for this start. |

**Walkover/retirement handling is deliberately unchanged.** Search-engine summaries claiming "TA excludes them" were shown to be hallucinations — no primary source says so — so we keep counting them (Sackmann's CSVs list a winner). Don't flip this without a primary source.

## Calibration & achieved accuracy

`ingest/calibrate-elo.ts` runs the engine forward to "today" (full 1968+ history, **dock ON**), scrapes the live TA board, joins by the **dominant-id** key (same as production), grid-searches the entrant seed to minimize median |overall error| over the top-50 TA players, and — the metric you actually need — **reports the single largest |deviation| and the worst-12 offenders**, not just the mean (a few injury-history players dominate the tail, so the mean hides them). Fitted 2026-06-15 vs TA as-of 2026-06-14:

| Tour | seedTour | seedSub | overall (median / meanAbs) | hard | clay | grass | worst |
|---|---|---|---|---|---|---|---|
| ATP | 1450 | 1130 | −3 / **15** | 17 | 20 | **96** | Djokovic +114 |
| WTA | 1350 | 1050 | +2 / **11** | 11 | 15 | 54 | Anisimova +112 |

The dock cut ATP meanAbs ~21→15 and **removed the absent-player outliers** (Alcaraz/Korda, formerly +90…+115, now ~±10). The **top-18** reproduce overall Elo within ~9 (ATP) / ~6 (WTA) median (the fixture). Three couplings: including Challengers while seeding high inflates the pool ~+260 Elo (the low-1200s reseed cancels it); extending the history start 2000→1968 inflated it ~+40 (re-absorbed by a ~−50 lower seed); and the dock lets the seed stop being dragged down to mask docked absentees. **Inclusion, history-start, seed, and dock all move together.**

## Per-individual residuals — what's left, and why

Per-individual (not mean) analysis shows the field is well-fit (most players ~5–7 Elo). After the injury/absence dock, the large deviations have these causes:

1. **Currently-absent players ARE docked — and now we dock them too.** TA's published board reduces an absent player's rating (verbatim, 2018: the penalties *"bring down the current Elo ratings of the players who are in the middle of long breaks … giving us ranking tables that come closer to what we expect"*; the COVID footnote shows he had to explicitly **suspend** it board-wide). We apply the documented curve (−100 @ 8 active-season weeks → −150 @ ~1yr) to players rated ≥1900. This maps Alcaraz/Korda (ATP) and leaves the sub-1900 WTA absentees (Kudermetova, Kartal, Danilovic) correctly un-docked — they fall below TA's ≥1900 gate. *(Earlier we wrongly concluded TA shows absentees frozen and removed the dock; the verbatim source + COVID suspension + the empirical 5/7-within-±10 fit overturned that — see findings §5.)*
2. **The on-return K-multiplier — the dominant *remaining* residual, and a bounded limit.** Currently-*active* injury returnees (Djokovic +114, Fritz +100; Anisimova +112, Zheng +95) are TA's largest board-wide deviations *after* the dock. TA depresses them via a **separate** mechanism — an on-return K-multiplier (×1.5 decaying to ×1 over 20 matches) applied through their actual post-return matches. An extraction-time dock structurally cannot reproduce a path-dependent K-schedule, and inflating the dock to chase active players would corrupt the source-faithful curve. The proper fix is to simulate that K-schedule in the engine (a separate, larger task); until then this is the documented ceiling.
3. **Vondrousova (−65, over-docked)** is the only ≥1900 WTA absentee the curve touches, and TA's single (ATP-derived) curve is too steep for her at 19 weeks. With N=1 above the gate, any "correction" is overfitting — and a per-gender scale was the unprincipled knob that got the dock wrongly removed before — so we accept her as the cost of faithfully carrying TA's one curve to both tours.
4. **ATP thin-grass samples** — large per-surface grass errors for players with few grass matches; small overall impact.

**Data sources:** Sackmann remains the single source of truth. atptour.com is Cloudflare-blocked; wtatennis.com's JSON API is open but its ToS forbids automated harvesting *and* it agrees with Sackmann to the day — so it adds legal/maintenance risk for zero gain (use only for occasional manual identity spot-checks, e.g. disambiguating Veronika vs Polina Kudermetova). Duplicate Sackmann `player_id`s (a maintainer-tracked multi-feed artifact, not a juniors/pro split) are handled by the dominant-id join.

## Re-deriving

```bash
npx tsx ingest/calibrate-elo.ts                 # re-fit the seed vs the live board (network)
# transcribe the winning seeds into ingest/elo-config.ts
pnpm backfill-elo && pnpm reindex               # recompute all snapshots (network), rebuild the index
ELO_FIXTURE=1 TZ=UTC npx vitest run ingest/historical-elo.fixture.test.ts   # opt-in regression guard
```

TA updates ~weekly, so the fixture reference (`ingest/fixtures/ta-elo-reference.json`) is pinned with its `asOf` date and re-captured deliberately via `.scratch/capture-ta-reference.ts`. Acceptance sanity check (issue #20): RG 2016 frozen → Djokovic #1 overall **and** #1 clay, Nadal #2 clay.

## Storage note

The frozen Elo is currently embedded per-snapshot. A planned normalized build-time store (`docs/superpowers/plans/2026-06-14-data-storage-normalization.md`) would lift it into a derived artifact and replay history once instead of per-snapshot — independent of this Elo work.
